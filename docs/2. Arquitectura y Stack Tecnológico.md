Aquí las preguntas son más ?de elección de herramientas? y deben estar alineadas con la experiencia del equipo de desarrollo y al tipo de producto.

### 1. Restricciones personales y del equipo

1. ¿De acuerdo a la experiencia del equipo qué tecnologías serían  las más apropiadas para esta aplicación específica?
    - Backend (Python, Laravel, Node, etc.)
    - Frontend (React, Vue, Svelte, etc.)
2. ¿El proyecto está pensado para que otros contribuyan fácilmente (open source)?
    - ¿Qué stack resultaría más amigable para colaboradores?

### 2. Frontend
1. ¿La web app será:
    - SPA (React, Vue) consumiendo una API, o
    - SSR/Blade (si usas Laravel), o un híbrido?
       
2. ¿Qué framework/componentes usarás para UI?
    - Tailwind, Bootstrap, otra.
3. ¿Necesitas PWA (uso ?tipo app? en el celular)?
4. ¿Cómo vas a gestionar el estado global (si usas SPA)?
    - Context, Zustand, Redux, Pinia, etc.
### 3. Backend

1. ¿Qué lenguaje y framework usarás para el backend?
    - Python por el uso de la IA, Laravel por experiencia u otros?
2. ¿Será una API REST limpia?
    - ¿Consideras GraphQL o no es necesario?
3. ¿Cómo vas a manejar:
    - Autenticación (JWT, sesión, tokens de API),
    - Autorización (roles, scopes, etc.)?

### 4. Base de datos y almacenamiento
1. ¿Qué motor de base de datos usarás?
    - PostgreSQL, MySQL/MariaDB, etc.
2. ¿Necesitas desde inicio:
    - Migraciones bien versionadas,
    - Seeds con datos de ejemplo para el demo?
3. ¿Dónde guardarás:
    - Logs,
    - Auditoría (quién creó/edición movimientos),
    - Metadatos de IA (ej. prompts, respuestas)?

### B.5. Stack de IA
1. ¿Qué proveedor(es) de IA vas a usar inicialmente?
    - Gemini, OpenAI, etc.
2. ¿Qué modelos específicos usarás para:
    - Clasificar gastos,
    - Generar resúmenes.
3. ¿Planeas tener una capa de abstracción para poder cambiar de proveedor (ej. hoy Gemini, mañana otro)?
4. ¿Consideras usar un LLM open source local en el futuro?
    - ¿Dejas esa puerta abierta en el diseño?

### 6. Infraestructura, DevOps y calidad
1. ¿Dónde desplegarás el backend?
    - VPS, servicios administrados, compartir hosting (no ideal), etc.
2. ¿Dónde desplegarás el frontend?
    - Vercel, Netlify, mismo servidor, etc.
3. ¿Vas a definir desde el inicio:
    - Pipeline CI/CD simple (tests + deploy),
    - Entorno de staging para probar antes de producción?
4. ¿Qué herramientas usarás para monitoreo y logs?
    - Log centralizado, alertas básicas, etc.
5. ¿Vas a incluir tests automáticos desde el MVP?
    - ¿Cuáles? (unitarios, de integración, al menos para el core financiero).
